Gessit

Projectbeschrijving

De casus die ik heb gemaakt/gekozen is het maken van een forumapplicatie genaamd Gessit. De applicatie is te vergelijken met bijvoorbeeld Reddit (vandaar de naam ;) ). Bezoekers kunnen namelijk binnen mijn site communiceren en discussiëren over allerlei verschillende soorten onderwerpen wanneer zij zich registeren als gebruiker binnen mijn app. Om deze communicatie gescheiden te houden en de app geordend te laten maak ik gebruik van zogenoemde Communities. Gebruikers kunnen dus berichten plaatsen binnen deze Communities die ook weer gemaakt kunnen worden door de gebruikers zelf om zo discussies die over hetzelfde onderwerp gaan bij elkaar te houden, zodat de applicatie geordend blijft en het voor de gebruiker overzichtelijker word. Binnen deze Communities kunnen ook weer Threads aangemaakt worden om zo een Community nog verder op te delen en georganiseerd te houden. Binnen deze Threads kunnen gebruikers berichten sturen en met elkaar communiceren. Op al deze verschillende entiteiten kunnen verscheidene operaties uitgevoerd worden binnen mijn app. Met deze site hoop ik uiteindelijk het stellen van vragen en het starten van communicatie te bevorderen tussen mensen over de gehele wereld om hen zo dichter bij elkaar te brengen, zodat ze altijd hulp kunnen vragen en nieuwe connecties kunnen maken binnen mijn app.

Entiteitbeschrijving

De entiteiten die ik heb gebruikt binnen mijn applicatie zijn de volgende:

  • User
  • Community
  • Theme
  • Thread
  • Message

Hieronder beschrijf ik alle verschillende bovenstaande entiteiten en hoe zij verband houden met elkaar.

De User entiteit weerspiegeld de geregistreerde gebruiker. Doormiddel van de User entiteit kunnen er nieuwe accounts gemaakt worden binnen de app en kan er authenticatie en authorizatie plaatsvinden. Daarnaast zijn vrijwel alle andere entiteiten gekoppeld aan de User entiteit, waardoor deze entiteit dus echt de fundering is van mijn applicatie en dus het belangrijkste is.

De Community entiteit weerspiegelen de verschillende "groepen" die je aan kan maken als geregistreerde gebruiker binnen de applicatie. Deze entiteit bevat ook weer andere entiteiten, zoals de Message en Thread entiteiten. De Community entiteit zorgt ervoor dat deze andere entiteiten gegroepeerd kunnen worden en dus gesplitst kunnen worden op onder andere basis van het thema en het onderwerp, hier komt de Theme entiteit bij kijken. Ook zorgt de Community entiteit er voor dat de applicatie gestructureerd en overzichtelijk blijft, omdat dus alle gerelateerde discussies en gesprekken gegroepeerd zijn binnen Communities. Een Community kan gemaakt worden door een geregistreerde gebruiker, maar er kan ook aan deel worden genomen door een ingelogde gebruiker wanneer deze Community gemaakt is door een andere gebruiker. Binnen een Community kunnen ook weer zogenoemde Thread entiteiten toegevoegd worden, wat eigenlijk een soort tekstkanaal is.

De Theme entiteit zorgt ervoor dat iedere Community entiteit over een bepaald thema kan beschikken, om zo deze entiteit meer een eigen "smaak" te geven. Gebruikers krijgen door het Theme entiteit nu namelijk een beter beeld van waar een bepaalde Community om draaid en wat de gespreksonderwerpen vooral gaan zijn, zodat zij zo een keuze kunnen maken of zij deel willen nemen aan een Community. De Theme entiteit zorgt er dus echt voor dat de Community entiteit meer een eigen doel gaat krijgen wat de gebruiker ook weer meer een reden geeft om meer dan één Community aan te maken, waardoor gebruikers dus geinteresseerd en actief blijven in mijn app.

De Thread entiteit zorgt ervoor dat gebruikers berichten kan sturen binnen een Community entiteit. De Thread entiteit is als het ware de "groep" binnen een Community waarin berichten gestuurd kunnen worden. De Thread beschikt over een titel en inhoud, waarmee de gebruiker een bepaalde vraag kan stellen of een discussie kan starten. Andere gebruikers kunnen hier dan weer op reageren door zogenoemde Message entiteiten aan te maken binnen die Thread. De Thread is dus eigenlijk het soort van tekstkanaal binnen een Community waar gebruikers berichten in aanmaken om zo hun mening te delen.

De Message entiteit zorgt ervoor dat gebruikers hun mening kunnen delen en vragen kunnen beantwoorden binnen een Thread. Message entiteiten zijn eigenlijk onderdeel van de fundatie van mijn app, omdat gebruikers doormiddel van deze Messages te maken dus kunnen communiceren met andere gebruikers om zo allerlei vragen te kunnen beantwoorden of discussies te starten wat uiteindelijk uiteraard het einddoel is van mijn site.

Entiteiten

In de onderstaande afbeelding is mijn ERD te zien die ik heb gemaakt voor mijn forum casus applicatie.

Hieronder beschijf ik alle entiteiten en attributen die ook in de bovenstaande ERD voorkomen met daarbij de bijbehorende datatypen.

User (de gebruiker zelf die berichten kan sturen, communities aan kan maken etc.)

  • id: ObjectId (“autogenerated” door mongodb)
  • joinedCommunities: ObjectId Array (verwijzingen naar Community entiteit)
  • createdCommunities: ObjectId Array (verwijzingen naar Community entiteit)
  • followers: ObjectId Array (verwijzingen naar User entiteit)
  • following: ObjectId Array (verwijzingen naar User entiteit)
  • roles: Role Array (enum: user/admin)
  • username: String
  • birthDate: Date
  • emailAddress: String
  • phoneNumber: String
  • password: String
  • isActive: Boolean
  • registerDate: Date
  • image: String

Community (de "groep" waarin Threads worden aangemaakt waarin berichten worden verstuurd)

  • id: ObjectId (“autogenerated” door mongodb)
  • owner: User (User entiteit)
  • members: ObjectId Array (verwijzingen naar User entiteiten)
  • threads: Thread Array (Thread entiteiten)
  • themes: Theme Array (Theme entiteiten)
  • name: String
  • description: String
  • ranking: Number
  • creationDate: Date
  • image: String
  • isOpen: Boolean

Theme (het thema van een Community)

  • id: ObjectId (“autogenerated” door mongodb)
  • name: String

Thread (een "chat" binnen een Community waar gebruikers berichten in kunnen sturen)

  • id: ObjectId (“autogenerated” door mongodb)
  • creator: ObjectId (verwijzing naar User entiteit)
  • messages: Message Array (Message entiteiten)
  • likes: ObjectId Array (verwijzingen naar User entiteit)
  • title: String
  • content: String
  • views: Number
  • likes: Number
  • creationDate: Date
  • image: String

Message (een bericht wat een gebruiker maakt en verstuurd)

  • id: ObjectId (“autogenerated” door mongodb)
  • creator: ObjectId (verwijzing naar User entiteit)
  • likes: ObjectId Array (verwijzingen naar User entiteit)
  • content: String
  • creationDate: Date
  • hasLikes: Boolean
