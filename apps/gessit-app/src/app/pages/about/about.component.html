Gessit

Projectbeschrijving

De casus die ik heb gemaakt/gekozen is het maken van een forumapplicatie genaamd Gessit. De applicatie is te vergelijken met bijvoorbeeld Reddit (vandaar de naam ;) ). Bezoekers kunnen namelijk binnen mijn site communiceren en discussiëren over allerlei verschillende soorten onderwerpen. Om deze communicatie gescheiden te houden en de app geordend te laten maak ik gebruik van zogenoemde "communities" die nu al in deze ontwikkelstage van de app te zien zijn en waarop al verschillende operaties op uitgevoerd kunnen worden. Gebruikers kunnen dus in de toekomst berichten plaatsen binnen deze "communities" die ook weer gemaakt kunnen worden door de gebruikers zelf om zo discussies die over hetzelfde onderwerp gaan bij elkaar te houden, zodat de applicatie geordend blijft en het voor de gebruiker overzichtelijker word. Binnen deze "communities" kunnen ook weer "threads" aangemaakt worden om zo een "community" nog verder op te delen en georganiseerd te houden. Binnen deze "threads" kunnen gebruikers in de toekomst berichten sturen en met elkaar communiceren. Op het "thread" object kunnen al verscheidene operaties uitgevoerd worden binnen mijn app op dit moment. Met deze site hoop ik uiteindelijk het stellen van vragen en het starten van communicatie te bevorderen tussen mensen over de gehele wereld om hen zo dichter bij elkaar te brengen, zodat ze altijd hulp kunnen vragen en nieuwe connecties kunnen maken binnen mijn app.

Entiteitbeschrijving

Één van de entiteiten die ik tot nu toe dus heb verwerkt is het zogenoemde "community" object. Dit is eigenlijk een soort van groep waar gebruikers in de toekomst lid van kunnen worden als zij bijvoorbeeld geinteresseerd zijn in een bepaald onderwerp waar dan die "community" de nadruk op legt en dat deze dus vol staat met discussies en vragen over dat specifieke onderwerp waar een gebruiker nou juist zo geinteresseerd in is. Zo kunnen mensen makkelijk en snel meedoen aan de communicatie online en nieuws ontvangen over onderwerpen wat hun ook echt interessant vinden. Door gebruik te maken van "communities" blijven online gesprekken binnen mijn app ook goed gescheiden van elkaar op basis van onderwerp, wat de overzichtelijkheid en bruikbaarheid van mijn app verbeterd.

Naast het "community" object heb ik ook nog een "thread" entiteit verwerkt binnen mijn applicatie. Een "thread" kan eigenlijk gezien worden als een groep in een "community" waarin daadwerkelijk berichten gestuurd kunnen worden door ingelogde gebruikers. Iedere gebruiker binnen een "community" kan een "thread" aanmaken en het wat inhoud geven zoals bijvoorbeeld een afbeelding of een titel om onder andere een discussie te starten of een vraag te stellen aan de andere gebruikers binnen de "community". Gebruikers kunnen dan in de toekomst dus binnen de aangemaakte "thread" berichten sturen als reactie op de inhoud van de "thread" om zo vragen te beantwoorden of mee te doen aan discussies. Door een "community" op te delen in "threads" hou ik iedere community overzichtelijk en zorg ik ervoor dat meerdere gebruikers bijvoorbeeld hun vragen kwijt kunnen in een "community", omdat er dus dankzij de "threads" meerdere soort van tekstkanalen zijn waar gebruikers met elkaar kunnen chatten in de toekomst doormiddel van de "message" entiteit. Deze entiteit word in de toekomst uiteraard gerealiseerd om dus de gebruikers daadwerkelijk de mogelijkheid te geven om met elkaar in gesprek te gaan.

Hieronder geef ik de verschillende attributen die het "community" object en het "thread" object tot nu toe hebben (met een in-memory database) en benoem ik de datatypes ervan en geef ik een korte beschrijving.

Het "community" object:
  • _id: string (de unieke identificator)
  • name: string (een naam)
  • description: string (een beschrijving)
  • image: string (de url)
  • creationDate: Date (wanneer hij gemaakt is)
  • isOpen: boolean (of hij publiek is)
Het "thread" object:
  • _id: string (de unieke identificator)
  • communityId: string (de unieke identificator van de gelinkte community)
  • title: string (de titel)
  • content: string (de inhoud)
  • views: number (hoe vaak hij bekeken is)
  • likes: number (hoe vaak hij "geliked" is)
  • dislikes: number (hoe vaak hij "gedisliked" is)
  • creationDate: Date (wanneer hij gemaakt is)
  • image: string (de url)

Entiteiten

In de onderstaande afbeelding is mijn ERD te zien die ik heb gemaakt voor mijn forum casus applicatie.

Hieronder beschijf ik alle entiteiten en attributen die ook in de bovenstaande ERD voorkomen met daarbij de bijbehorende datatypen.

User (de gebruiker zelf die berichten kan sturen, communities aan kan maken etc.)

  • id: ObjectId (“autogenerated” door mongodb)
  • name: String
  • birthDate: Date
  • emailaddress: String
  • phonenumber: String
  • password: String
  • registerDate: Date
  • image: String

Community (de "groep" waarin "threads" worden aangemaakt waarin berichten worden verstuurd)

  • id: ObjectId (“autogenerated” door mongodb)
  • name: String
  • description: String
  • ranking: Number
  • creationDate: Date
  • image: String
  • isOpen: Boolean

Theme (het thema van een "thread" of "community")

  • id: ObjectId (“autogenerated” door mongodb)
  • name: String

Thread (een "chat" binnen een "community" waar gebruikers berichten in kunnen sturen)

  • id: ObjectId (“autogenerated” door mongodb)
  • title: String
  • content: String
  • views: Number
  • likes: Number
  • dislikes: Number
  • creationDate: Date
  • image: String

Message (een bericht wat een gebruiker maakt en verstuurd)

  • id: ObjectId (“autogenerated” door mongodb)
  • content: String
  • likes: Number
  • dislikes: Number
  • sendDate: Date
